# 机制合理性修复方案

## 背景

当前游戏基本内容已完善，但多项机制存在不合理处，影响游戏体验和数值平衡。本文档系统梳理所有不合理机制点，给出修复方案和优先级。

---

## P0 — 破坏游戏循环 / 数值失控

### P0-1：战斗掉落物品固定为 `spirit_stone`，无视敌人类型

**位置：** `src/lib/combat-engine.ts` 第 80 行

```typescript
if (Math.random() < luckRate) items.push("spirit_stone");
```

**问题：** 所有敌人无论类型（野狗、妖兽、邪修），掉落物品都是 `spirit_stone`，毫无差异化。

**修复：** 按敌人类型配置掉落池：

```
野狗 → 灵石 + 低概率掉落「兽骨」
妖兽 → 灵石 + 中概率掉落「妖丹」
邪修 → 灵石 + 中概率掉落「法器碎片」
BOSS → 灵石 + 高概率掉落「功法残页」
```

## P1 — 明显不合理但可继续游玩

### P1-1：行动和商店不随境界解锁

**位置：** `src/lib/cultivation-data.ts` — `ACTIONS` 数组、`SHOP_ITEMS`

**问题：** 从炼气期到渡劫期，玩家可用的行动完全一样（9 个固定行动），商店物品也完全一样。没有"突破后解锁新内容"的成长感。

**修复：**

**行动：** `Action` 接口增加 `minRealm?: string` 字段：

```
炼气期 → 解锁：打坐修炼、吐纳练气、外出历练
筑基期 → 解锁：炼丹制药、洞府闭关
结丹期 → 解锁：神识探宝
```

`getAvailableActions` 增加 `realm` 过滤。

**商店：** `ShopItem` 增加 `minRealm?: string` 字段：

```
凡人/炼气 → 益气丹、木剑
筑基 → 锻骨丹、铁剑、灵剑
结丹+ → 破境丹、法袍、灵珠手串
```

**坊市：** 修仙交易市场改为 17 岁解锁（觉醒后第一年适应期）。

**夺宝机制：** 练气期玩家可以在坊市购买超出自身境界的物品，但如果离开坊市时未使用遮掩手段，有概率触发「夺宝」战斗事件（敌人战力 = 物品价值 × 系数）。

### P1-2：功法熟练度获取太慢

**位置：** `src/app/api/narrative/route.ts` 第 147 行

```typescript
const result = addProficiency(r.level, r.proficiency, t.upgradeProficiency, Math.floor(Math.random() * 6) + 5);
```

**问题：** 每次行动只获得 5-10 熟练度，升到 2 级需要 100-150（10-30 次行动），升到 3 级需要 300-400（40-80 次行动）。玩家可能直到道消都升不满一门功法。

**修复：** 熟练度获取途径扩展为三种：

| 途径 | 说明 |
|------|------|
| 日常修炼 | 每次行动获得 `10 + 境界索引 × 5` 熟练度 |
| 战斗使用 | 每次战斗胜利，装备的功法获得 `15 + 境界索引 × 5` 熟练度 |
| 主动参悟 | 新增「参悟功法」行动，消耗 5 体力获得 `20 + 境界索引 × 5` 熟练度 |

### P1-3：战斗叙事模板化，无 AI 生成

**位置：** `src/lib/combat-engine.ts` 第 103-112 行

**问题：** `getCombatNarrativeText` 是硬编码的 5 条模板文案，每次战斗看到相同文本。虽然尝试调用 `generateCombatNarrative`，但该函数本身也是模板 fallback，并未真正调用 AI。

**修复：** 在 `narrative.ts` 中实现真正的 `generateCombatNarrative`，调用 `callAI` 按实力差距生成差异化战斗叙事：

```
我方远强于敌人 → 风格：轻松碾压
实力相当 → 风格：惊险激战
我方远弱于敌人 → 风格：紧张搏命
```

当前模板作为 AI 失败时的 fallback。

---

### P1-4：关系系统缺少距离和行动力限制

**位置：** `src/app/relationships/page.tsx`、`src/lib/cultivation-data.ts`

**问题：** 当前关系页面可以随时与任何人交流，没有距离和体力限制，不合理。

**修复：**

- 关系页列出所有有过交往的人物（NPC + 家庭成员）
- 交流需要满足两个条件：
  1. 对方在当前所在地点（或通过手机联系）
  2. 消耗 2 点行动力
- 没有手机时只能与附近的人当面交流
- 手机解锁后可以远程发消息（仍需消耗 1 行动力）

### P1-5：心魔系统

**位置：** **新建** — `src/lib/mind-demon.ts`、`prisma/schema.prisma`

**问题：** 当前没有任何心魔机制。道心受损 debuff 只影响修炼效率，没有修仙小说中心魔的独特体验。

**修复：**

**数据结构：** `Cultivator` 新增 `mindDemon Int @default(0)` 字段，表示心魔值（0-100）。

**触发条件：**
| 事件 | 心魔增加值 |
|------|-----------|
| 道心受损（战斗喜剧翻车） | +10 |
| 突破失败 | +15 |
| 杀戮过多（连续战斗胜利） | +5/次 |
| 道消轮回 | 清零 |

**效果：** 心魔值 ≥ 50 时，每次行动有 `心魔值 / 2`% 概率触发心魔事件——AI 生成一段心魔叙事，扣除额外体力或经验。

**解除方式：**
| 方式 | 效果 |
|------|------|
| 使用愈灵符 | -20 |
| 静修闭关（SECLUSION 行动） | -5/次 |
| 突破境界 | -30 |
| 轮回转世 | 清零 |

## P2 — 体验优化 / 平衡性调整

### P2-1：受伤 UI 显示

**位置：** `src/app/dashboard/page.tsx`

**问题：** 玩家受伤后 `injuryDebuff` 在后台生效，但界面上没有任何提示。

**修复：** Dashboard 角色信息区显示当前伤势状态图标。鼠标悬浮时弹出详细描述：

```
🔴 重伤（剩余 3 次行动）
  修炼效率减半，突破概率减半
```

伤势等级按 `injuryDebuff` 值区分：

| injuryDebuff | 显示 | 效果 |
|-------------|------|------|
| 0 | 无 | 正常 |
| 1 | 🟡 轻伤 | 修炼效率 ×0.8 |
| 2-3 | 🔴 重伤 | 修炼效率 ×0.5 |
| 4+ | ⚫ 濒死 | 修炼效率 ×0.3 |

---

### P2-2：道心受损改为心魔触发

**位置：** `src/lib/combat-engine.ts` 第 94 行

**问题：** 当前档 0 惩罚（`ratio < 1` 的喜剧翻车）设置 `injuryDebuff: 1`，但文案说是"道心受损"（悟性-2），实际效果与文案不符。

**修复：**

- 取消 `injuryDebuff` 与"道心受损"的绑定
- 档 0 惩罚改为：丢灵石 + `mindDemon += 10`（触发心魔系统 P1-5）
- 档 0 惩罚命名改为"道心受损"而非"轻伤"

---

### P2-3：战斗失败物品丢失

**位置：** `src/lib/combat-engine.ts` 第 91-98 行

**问题：** 档 1 惩罚返回 `itemLoss: []`，空数组表示没有实际丢失物品。设计文档要求丢 1-2 件物品，但代码未实现。

**修复：** 在 `applyPenalty` 中实现物品丢失逻辑：

- 档 1（`ratio < 2`）：从背包中随机选取 1 件非装备物品移除
- 档 2（`ratio < 5`）：从背包中随机选取 2 件非装备物品移除
- 档 3（`ratio ≥ 5`）：道消，所有物品丢失

在 `action/route.ts` 中处理 `itemLoss` 数组，更新背包 `inventory` JSON。

---

### P2-4：觉醒逻辑合并到 advance-year

**位置：** `src/app/api/action/route.ts` 第 45-48 行

**问题：** 觉醒条件在行动路由和年份推进路由中各写了一份。虽然 realm 持久化后不会重复触发，但逻辑重复、维护成本高。

**修复：** 删除 `action/route.ts` 中的觉醒检查代码（第 45-48 行），觉醒统一在 `advance-year/route.ts` 中处理。
