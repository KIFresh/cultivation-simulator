# 机制合理性修复方案

## 背景

当前游戏基本内容已完善，但多项机制存在不合理处，影响游戏体验和数值平衡。本文档系统梳理所有不合理机制点，给出修复方案和优先级。

---

## P0 — 破坏游戏循环 / 数值失控

### P0-1：年份推进无消耗，可无限连点跳过游戏内容

**位置：** `src/app/api/advance-year/route.ts`

**问题：** 推进年份不消耗任何资源（体力、金币），玩家可以连续点击推进年份，跳过所有日常行动、战斗、奇遇等内容，直接衰老到道消。

**修复：** 推进年份消耗 5 体力 + 20 金币。体力不足时阻止推进。

---

### P0-2：体力没有自然恢复机制

**位置：** `src/lib/cultivation-data.ts` — `calculateMaxStamina`、`src/app/dashboard/page.tsx`

**问题：** 体力只能通过推进年份恢复。玩家用完体力后只能推进年份，再次触发了 P0-1 的无限连点问题。

**修复：** 每 5 分钟自动恢复 1 点体力（上限不超过最大体力值）。在 `dashboard` 或服务端定时检查。

---

### P0-3：战斗掉落物品固定为 `spirit_stone`，无视敌人类型

**位置：** `src/lib/combat-engine.ts` 第 80 行

```typescript
if (Math.random() < luckRate) items.push("spirit_stone");
```

**问题：** 所有敌人无论类型（野狗、妖兽、邪修），掉落物品都是 `spirit_stone`，毫无差异化。

**修复：** 按敌人类型配置掉落池：

```
野狗 → 灵石 + 低概率掉落「兽骨」
妖兽 → 灵石 + 中概率掉落「妖丹」
邪修 → 灵石 + 中概率掉落「法器碎片」
BOSS → 灵石 + 高概率掉落「功法残页」
```

## P1 — 明显不合理但可继续游玩

### P1-1：行动不随境界解锁，所有阶段行动相同

**位置：** `src/lib/cultivation-data.ts` — `ACTIONS` 数组

**问题：** 从炼气期到渡劫期，玩家可用的行动完全一样（9 个固定行动）。没有"突破后解锁新行动"的成长感。

**修复：** `Action` 接口增加 `minRealm?: string` 字段：

```
炼气期 → 解锁：打坐修炼、吐纳练气、外出历练
筑基期 → 解锁：炼丹制药、洞府闭关
结丹期 → 解锁：神识探宝
```

`getAvailableActions` 增加 `realm` 过滤。

---

### P1-2：商店物品不随境界变化

**位置：** `src/lib/cultivation-data.ts` — `SHOP_ITEMS`

**问题：** 商店物品固定，不随玩家境界变化。炼气期和渡劫期玩家看到一样的商品。

**修复：** `ShopItem` 增加 `minRealm?: string` 字段，`getShopItems` 按玩家境界过滤：

```
凡人/炼气 → 益气丹、木剑
筑基 → 锻骨丹、铁剑、灵剑
结丹+ → 破境丹、法袍、灵珠手串
```

---

### P1-3：功法熟练度获取太慢

**位置：** `src/app/api/narrative/route.ts` 第 147 行

```typescript
const result = addProficiency(r.level, r.proficiency, t.upgradeProficiency, Math.floor(Math.random() * 6) + 5);
```

**问题：** 每次行动只获得 5-10 熟练度，升到 2 级需要 100-150，即 10-30 次行动。升到 3 级需要 300-400，即 40-80 次行动。玩家可能直到道消都升不满一门功法。

**修复：** 熟练度改为 `10 + 境界索引 × 5`，高境界玩家获得更快：

```
炼气期 = 10+1×5 = 15/次
筑基期 = 10+2×5 = 20/次
结丹期 = 10+3×5 = 25/次
...
```

---

### P1-4：战斗叙事模板化，无 AI 生成

**位置：** `src/lib/combat-engine.ts` 第 103-112 行

**问题：** `getCombatNarrativeText` 是硬编码的 5 条模板文案，每次战斗看到相同文本。虽然第 143 行尝试调用 `generateCombatNarrative`，但该函数本身也是模板 fallback，并未真正调用 AI。

**修复：** 在 `narrative.ts` 中实现真正的 `generateCombatNarrative`，调用 `callAI` 生成差异化战斗叙事。当前模板作为 AI 失败时的 fallback。

---

### P1-5：NPC 仅有 5 个且位置固定，无互动深度

**位置：** `src/lib/cultivation-data.ts` 第 298-312 行

**问题：** NPC 列表仅 5 人，位置固定（home/school/downtown/market），缺乏互动深度。NPC 不会根据玩家境界变化，不会给任务，不会成长。

**修复：** 增加 NPC 数量至 15-20 个，分布在所有地点。NPC 对话增加好感度门槛和境界要求。

---

## P2 — 体验优化 / 平衡性调整

### P2-1：`injuryDebuff` 缺乏 UI 显示

**位置：** `src/app/dashboard/page.tsx`

**问题：** 玩家受伤后 `injuryDebuff` 在后台生效（修炼效率减半），但界面上没有任何提示。玩家不知道自己受伤了。

**修复：** Dashboard 角色信息区显示受伤状态图标和剩余次数。

---

### P2-2：道心受损 debuff 无实际效果

**位置：** `src/lib/combat-engine.ts` 第 94 行

```typescript
if (ratio < 1) return { goldLoss: ..., injuryDebuff: 1, ... };
```

**问题：** 档 0 惩罚设置了 `injuryDebuff: 1` 但文案说这是"道心受损"（悟性-2），实际 `injuryDebuff` 只降低修炼效率，没有悟性-2 的效果。

**修复：** 新增 `mindDebuff` 字段，或修改 `injuryDebuff` 文案为"轻伤"。

---

### P2-3：战斗失败物品丢失未实现

**位置：** `src/lib/combat-engine.ts` 第 91-98 行

**问题：** 档 1 惩罚（`ratio < 2`）返回 `itemLoss: []`，空数组表示没有实际丢失物品。设计文档要求丢 1-2 件物品，但代码未实现。

**修复：** 实现物品丢失逻辑：从背包中随机选取 1-2 件非装备物品移除。

---

### P2-4：觉醒事件在行动和年份推进中重复触发

**位置：** `src/app/api/action/route.ts` 第 45-48 行、`src/app/api/advance-year/route.ts` 第 84 行

**问题：** 觉醒条件在两个地方都写了，但 `cultivator.age >= 16` 在行动路由中每次都会检查，虽然 realm 持久化后不会重复触发，但逻辑上应该统一到一处。

**修复：** 觉醒统一在 `advance-year` 路由中处理，行动路由不再检查觉醒。
